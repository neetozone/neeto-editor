# neeto-editor v2 (neeto-atoms)

These stories render the **v2 editor entry** (`@bigbinary/neeto-editor/v2`),
which is built on `@bigbinary/neeto-atoms` (shadcn-based) instead of
`@bigbinary/neetoui`. The v1 stories under "Examples", "Walkthroughs", etc.
continue to render the original neetoUI-based editor.

> v1 and v2 ship side-by-side from the same npm package. Consumer products
> migrate at their own pace by switching their import path from
> `@bigbinary/neeto-editor` to `@bigbinary/neeto-editor/v2`. They never coexist
> on the same page.

## Known caveats while migration is in flight

- v2 entry currently re-exports the v1 `Editor`, `EditorContent`, `Menu`, and
  `FormikEditor` because those parents still mount v1 leaf components. As Wave
  4+ moves more parents over, these stories will progressively switch to
  pure-v2 internals.
- Tippy.js still drives the editor's in-canvas popups (BubbleMenu, slash
  command, image menu). That is intentional — the move to Floating UI is a
  separate post-migration project.
- Theming (CSS variables, dark mode polish) lands in Wave 6.
